# PDF原生结构目录识别与切片系统

## 设计思路

直接从PDF页面提取文字结构，绕过TXT转换的乱码和布局失真问题。

### 核心流程

1. **提取文字块** → 带 `(x, y, font, size, flags)` 的原始信息
2. **检测目录页** → 多特征评分（关键词+点号线+页码+缩进+章节模式）
3. **解析目录结构** → 提取标题、页码、层级（通过缩进+标题前缀）
4. **计算章节边界** → 基于目录页码映射，带边界保护
5. **验证章节位置** → 多策略匹配确认标题在对应页面出现
6. **切片输出** → 按章节切分PDF页面范围

### 关键优势

- 不依赖OCR，不损失排版信息
- 页码直接从PDF元数据获取，不会错乱
- 章节标题的字体/大小/位置是强信号，比TXT里的纯文本更可靠
- 对 `(cid:xx)` 乱码免疫——直接从PDF文字流提取，不经过中间格式

## 使用

```bash
python pdf_toc_slicer.py <pdf_path>
```
